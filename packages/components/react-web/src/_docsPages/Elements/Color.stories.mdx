# Color

Color is an integral aspect of conveying information in a design. Judicious use
of color is a great way to distinguish the Compassion brand and plays an
important role in creating consistent neighbour experiences.

Use color to augment clear communication, to indicate that elements are
interactive, to reinforce an information architecture, or to clarify how
elements on the screen relate to each other.

## Accessibility

There are two important accessibility guidelines related to color:

1. Always use a text color that provides sufficient contrast
   ([4.5:1 or greater](https://www.w3.org/TR/2008/REC-WCAG20-20081211/#visual-audio-contrast-contrast))
   with its background. Pragramtically, this means never using a gray lighter
   than the CDS’s “base gray” against a white background
2. Never use color along to indicate a condition or state; consider using an
   icon or more descriptive text in such cases

## Usage

Colors are provided via the `color` object in the `@compassion-gds/elements`
package:

```js
import { color } from '@compassion-gds/elements';
```

The `color` object exposes Compassion’s brand-aligned design tokens for digital
design. Using the `color` object instead of raw hex values means that any time
one of these values changes upstream, the proper value will always be provided
to your website or app. _Use the color object for all but the rarest cases!_

import { helpers, color } from '@compassion-gds/elements';

export const ColorSwatch = styled.div`
  display: inline-grid;
  font-family: 'IBM Plex Mono';
  font-size: 0.625rem;
  grid-gap: 1rem;
  align-items: center;
  justify-items: center;
  text-align: center;
  &::before {
    content: '';
    display: block;
    height: 8rem;
    width: 8rem;
    border-radius: 50%;
    background: ${(props) => props.color};
  }
`;
